# 系統需求文件 (requirements.md)

## 1. 專案簡介
這是一個簡單的「圖書管理系統」API，旨在提供使用者透過網頁 API 查詢、新增、修改與刪除書單。此專案將作為 Web 架構、資料庫串接與雲端部署的實作練習。

## 2. 目標使用者
* **學校課堂夥伴 / 程式新手**：需要一個直覺、好上手的 API 範例來理解後端與資料庫的運作。
* **前端開發者**：能直接呼叫此 API 取得 JSON 格式的書單資料。

## 3. 核心功能需求
* **功能 A：查詢書單 (Read)**
  * 使用者可以讀取所有書籍的列表。
  * 每本書籍應包含：ID、書名、作者、出版年份。
* **功能 B：新增書籍 (Create)**
  * 使用者可以透過 API 輸入書名、作者及出版年份，並將其永久儲存至 PostgreSQL 資料庫中。
* **功能 C：刪除書籍 (Delete)**
  * 使用者可以指定特定的書籍 ID，將該書籍從資料庫中移除。

## 4. 非功能性需求
* **部署平台**：後端程式碼部署於 Render 雲端平台。
* **資料儲存**：使用 PostgreSQL 進行雲端或本地資料持久化。
* **API 文件**：需提供互動式的自動化 API 測試文件 (Swagger / OpenAPI)。
